Unidex

Thread safe, serializable, data structure; designed in a portable class dot net library, with variable number of columns per row within the same table. Use the N.U.O.T.A. (None Unidex Object Type Association) interfaces to quickly fill in your view model, and includes automatic OneDrive management for cross device/platform management of data.

Not just another database.

What can it do?

This is a completely different mindset that can form to work like any system, such as SQL, with tables, rows, columns, queries, or break free from tradition and create your own structure.

But why would I use it?

Besides the hardware compatibility of every device that supports the dot net portable library, or that the structure you inherit from in your code is both the memory model and the file structure, or that it automatically creates memory locations and file locations automatically just by writing the field property, which is just like code first structures for SQL, or even that it is truly 64 bit with absolute response time and handles over 200 GB of ram for a single application, use it to simplify your life.
